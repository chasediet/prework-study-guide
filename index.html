Git

  • Git is used for task management within coding
  • You have git locally and then git remotely
  • Git status- helps you check which branch you are in and shows any files that have been modified or changed in the working directory, staged for your next commit
  • Git checkout- switches you to another branch and makes it your working directory?
  • git pull fetches and merges any commits from the tracking remote branch
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • What the different letter abbreviations mean
    1. -b: Example: git branch -b <new_branch> Used to create a new branch and then switch to it. So basically without -b you would have to do git branch <new_branch> and then git checkout <new_branch>
    2. -A: Example: git add -A: This will stage all modified and deleted files, as well as any new files that have been added to the working tree. Basically stages all changes, including new modified and deleted files. This command will stage all changes in the currenty directory and its subdirectories.
    3. -m: Example: git commit -m "added code to HTML file": The "m" option is used to specify a commit message when committing changes to the repository. It allows you to specify the commit message direcly on the command line instead of opening an editor to write the message.
